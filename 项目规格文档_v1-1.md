# 智能视觉小说编辑器 - 项目规格文档 v1

## 一、项目定位

一个桌面端应用，让不会写代码的创作者通过"中文剧本 + 手动摆放素材"的方式，制作出可在 Ren'Py 引擎中运行的视觉小说游戏。未来将接入大模型，实现"用户描述剧情 → AI自动理解并配置画面/音频"。

**技术栈：Electron + React + TypeScript + Zustand + Tailwind CSS**

**核心原则：不管是人工操作还是未来AI操作，最终都产出同一套结构化数据（Delta），再由确定性的系统逻辑转换成 Ren'Py 代码，AI不允许直接生成画面坐标或代码本身。**

---

## 二、核心数据结构：Delta（差量指令）

系统采用"状态归约"模型：

```
当前完整状态(S_i) = 合并(上一行完整状态(S_i-1), 本行差量指令(Δ_i))
```

每一行台词只记录"相对上一行发生了什么改变"，未声明的字段视为"继承上一行"，而不是重新计算。

### 2.1 位置槽位系统（解决画面抖动的核心机制）

```typescript
// 预定义命名槽位，禁止使用自由浮点坐标
type PositionSlot = {
  id: string;              // 如 "center", "left", "right_close"
  anchor_x: number;        // 归一化坐标 0-1
  anchor_y: number;
  anchor_point: "bottom" | "center"; // 对齐基准点
};

// 角色在某一行的状态引用的是槽位ID，不是坐标数值
// 若某行未对该角色下达任何指令，直接复用上一行的 position_slot 引用
```

### 2.2 单行 Delta 结构

```typescript
interface LineDelta {
  line_id: string;
  speaker: string | null;
  dialogue: string;

  background: {
    asset_id: string;
    transition?: string; // 如 "dissolve", "fade"
  } | null; // null = 继承上一行，不重新计算

  characters: {
    [characterId: string]: {
      sprite_id: string;         // 立绘/表情差分素材ID
      position_slot: string;     // 引用槽位ID
      transition?: string;
      action: "show" | "hide" | "__CLEAR__"; // 显式动作，不可省略语义
    }
  };

  audio: {
    bgm: AudioTrackInstruction | null;      // null=继承, "__CLEAR__"=显式停止
    ambient: AudioTrackInstruction | null;
    se: string[];          // 一次性音效，不继承
    voice: string | null;  // 绑定本行的语音文件
  };

  ai_meta?: {
    confidence: number;
    needs_review: boolean;
    source_text_span: [number, number];
  };
}

interface AudioTrackInstruction {
  asset_id: string;
  volume: number;      // 0-1
  fade_in_ms?: number;
  fade_out_ms?: number;
  loop: boolean;
}
```

**关键规则：`null` 表示"不修改该字段，继承上一行"；`"__CLEAR__"` 表示"显式清除/停止"。两者语义严格区分，不可混用，这是防止状态误判的核心约束。**

### 2.3 各轨道继承规则

| 轨道 | 继承行为 |
|---|---|
| 背景 | 持续继承，直到显式切换指令 |
| 角色立绘/位置/表情 | 持续继承，直到该角色被移动/换表情/退场 |
| BGM / 环境音 | 持续继承+循环，直到显式停止或替换 |
| 音效(SE) | 一次性事件，不进入继承链 |
| 语音 | 绑定单行的一次性事件，不继承 |

## 三、界面结构（四层布局）

```
┌────┬──────────┬────────────────────┬─────────┐
│最左 │  素材库   │      舞台预览        │ 剧本流   │
│侧边 │(背景/立绘/│   (核心视觉焦点，    │ (抽屉，  │
│栏  │ 音频)    │  所见即所得实时渲染)  │ 可钉住)  │
├────┴──────────┴────────────────────┴─────────┤
│         底部：多轨道时间轴                       │
│  背景轨/角色轨/BGM/环境音/SE/语音，色块=延续区间    │
└───────────────────────────────────────────────┘
```

- **最左侧边栏**：项目级导航（章节/场景树、角色库管理、导出设置、未来AI功能入口），折叠式图标栏
- **素材库**：Tab切换背景/立绘/表情差分/音频，拖拽到舞台或时间轴
- **舞台预览**：始终渲染"当前选中行"的合并后完整状态，是全局视觉锚点
- **剧本流**：抽屉式，默认收起/半展开浮层，支持"钉住"变为常驻侧栏；逐行编辑，选中行联动预览做交叉淡入淡出过渡
- **时间轴**：横向多轨道，色块横跨的行数=该状态延续的区间，直观呈现继承逻辑

**联动核心**：任意区域操作都围绕"当前选中行"同步，改动一处，其余区域立即反映。

## 四、开发阶段划分（AI应严格按此顺序推进，不要跳阶段）

**阶段一：数据核心（无UI）**
- 完成上述 TypeScript 类型定义
- 实现状态归约纯函数：输入 LineDelta[]，输出每行的完整合并状态
- 编写单元测试，覆盖：继承、显式覆盖、`__CLEAR__`清除、槽位复用、音频区间跨行延续等场景

**阶段二：骨架UI（假数据）**
- 按第三节的四层布局搭建界面骨架
- 使用写死的假数据渲染，暂不接入真实拖拽/文件系统
- 验证布局比例、抽屉交互、联动动画是否符合预期

**阶段三：真实交互**
- 素材库真实拖拽写入Delta状态
- 时间轴色块可拖拽调整区间起止行
- 预览真实渲染阶段一算出的合并状态
- 实现 Delta → 合法 `.rpy` 文件的导出逻辑

**阶段四：接入AI（此阶段暂不启动）**
- 编写系统提示词，教AI理解本文档定义的Delta Schema
- AI输出限定为"Draft Delta"，必须经校验层（槽位/素材ID合法性检查）确认后才可生效
- AI不允许直接输出坐标数值或Ren'Py代码
