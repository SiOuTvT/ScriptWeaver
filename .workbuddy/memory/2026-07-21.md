## ScriptWeaver 全站代码诊断（只读体检，未改代码）

受命作为首席架构师对 ScriptWeaver（React+Electron+Zustand+Ren'Py Codegen 可视化 Galgame 引擎，src/ 约 23.6k 行）做全站深读，产出《ScriptWeaver 全站诊断报告.md》。五维并行深读 + 关键 🔴 项源码复核。

### 关键结论（已写报告）
- **P0 严重**：
  1. `evalCondition`(varRuntime.ts:53-73) 沙箱逃逸——白名单放行 `(`/`=`，`localStorage.clear()` 真执行。
  2. `reducer.applyDelta`(reducer.ts:89-93) 加载缺 `delta.audio` 即崩；`loadProjectData` 无校验。
  3. Ren'Py 导出变量名未消毒(rpyExporter.ts:336/1178)→非法 `default`/`$`；`displayName` 与 `escapeDialogue` 漏转义 `\n`(108-110)；`BUILTIN_TRANSITIONS`(118-130) 含非法/错名(glitter/squeeze*/facin/facout 非法；moveinup→moveintop 等方向后缀错)。*复核 Ren'Py 官方文档纠正子审查：flash/vpunch/hpunch 是真实内置 transition，保留。*
  4. `deleteAsset`(appStore.ts:331-332) 从不 `revokeObjectURL(blobUrl)` → Web 模式 blob 泄漏。
- **P1 体验**：变量监视 Dock 无法收成 rail(常驻288px, AppLayout.tsx:443)；右侧三 Dock 堆叠挤压舞台无 min-width。
- **P2 健壮**：AI `cleanup()` 死代码未调用致 180s 定时器泄漏(aiDirector.ts:846-849)；`parseDirective` 不容忍尾逗号/无部分恢复；模态无 focus trap。
- **良好**：Store 单一数据源(resolvedStates 同 set 派生)、流式 sw-asset 协议零 base64 驻留、overlay portal 架构、断流 30s/180s 看门狗、AI 单事务提交。

### 共同根因
多处"信任上游 UI 输入 / 信任外部文件"假设，缺边界校验层。

## 0.5.5 战役 · 第一阶段 P0 缺陷消杀（已落地 + 验收通过）

用户批准诊断报告，正式开启「0.5.5 专项修复与 UI 完全重构战役」。第一阶段 P0 已全部完成并通过 tsc + 单测。

### P0-1 Ren'Py 导出消毒（rpyExporter.ts）
- `escapeDialogue` 现剔除 `\r` 并将 `\n`→`\\n`（修复台词字面换行断裂）。
- 新增 `PY_IDENT_RE` / `PYTHON_KEYWORDS` / `pyVarName()` 助手；`varOpExpr` 与 `default` 发射均经 `pyVarName` 消毒变量名。
- Character `define` 中 `displayName` 与 `dialogueColor` 均经 `escapeDialogue` 转义双引号。
- `BUILTIN_TRANSITIONS` 按 Ren'Py 官方手册纠正：移除 glitter/squeeze*/facin/facout；方向后缀 up/down→top/bottom（moveintop 等）；补 squares/slideaway*/zoomin*/ease 全系。
- `validateExportNames` 新增第 5 参 `variables`，校验 GlobalVariable.name 合法性（非法/关键字→报错提示并自动清洗）。
- 同步修正 `rpyExporter.test.ts` 内置过渡集合以匹配新白名单。

### P0-2 加载防护与沙箱安全（varRuntime.ts + reducer.ts）
- `evalCondition` 重写为 tokenizer + `ConditionParser` 递归下降求值器，彻底移除 `new Function` 沙箱逃逸（`.`/`;`/函数调用均报错 fail-open 返回 true，兼容历史行为）。
- `reducer.applyDelta`：`delta.background != null`、`characters ?? {}`、audio 整段用 `delta.audio ?? {}` 防御；新增 `normalizeDelta` 边界补全助手（纯函数、幂等）。
- `loadProjectData` 经 `normalizeDelta` 归一化并给 assets/characterConfigs/variables 默认值。

### P0-3 资产 blobUrl 泄漏（appStore.ts）
- 新增 `revokeAssetBlobs(assets)`；在 `deleteAsset`/`newProject`/`setAssets`/`loadProjectData` 切换前回收旧 blobUrl；接口新增 `disposeAllBlobs`。

### 验收结果
- `tsc --noEmit`：EXIT 0，零类型错误。
- `vitest run`：6 文件 / 90 用例全部通过（EXIT 0）。

## 0.5.5 战役 · 第二阶段 UI/UX 工业级完全重构（已落地 + 验收通过）

用户批准 P0，正式下达「第二阶段：UI/UX 工业级完全重构」，要求「完全推倒式重构」（非旧 DOM/CSS 打补丁）。调用 frontend-dev Skill 执行；保持既有 React18+Electron31+Tailwind v3+Zustand4+lucide-react 技术栈（**未引入 framer-motion/gsap**，Electron 打包有风险且 package.json 无此依赖），动效全部用 CSS transform + 项目已有的 `animate-slide-up`（GPU 友好、60fps、尊重 prefers-reduced-motion）。

### 核心结构重构：AppLayout 主界面（真正的"拆掉重建"）
- **新增 `src/components/layout/OverlayDrawer.tsx`**（ cornerstone）：浮层抽屉，absolute + `transform: translateX()` 滑入/滑出，默认关闭时**不占布局空间**→ 舞台+时间轴拉满。左缘 pointer 拖拽手柄自由收缩宽度（minWidth 220 / maxWidth 540）。`headerless` 模式专供变量监视（自带头部，改用悬浮 X 钮）。仅 transform/opacity 动画。
- **重写 `AppLayout.tsx`**：移除右侧三个 inline `<Dock>` 兄弟节点（原 248+288+320px 各自挤压舞台，根因）。改为：
  - `panels` 状态（script/vars/choice 默认**全 false** → 舞台满宽）。
  - `drawerRight(key)` 按 `['choice','vars','script']` 累加已开宽度，多抽屉从右向左**堆叠偏移**。
  - 44px 右缘 `<nav>` 工具栏 `RailToggle`（激活态 `bg-signal/15 text-signal`），一键唤出对应浮层抽屉。
- 左 `<Dock>`（素材库）与底部时间轴 Dock（可折叠）逻辑保留不变。

### 三大辅助页「视觉拉升」（逻辑已满足需求，改为工业级视觉/布局重构，非补丁式）
- **ScriptOverview.tsx**：仪表盘由 `divide-x` 条带改为 `grid-cols-2 sm:3 lg:6` **Bento 六卡**（圆角 2xl、`animate-slide-up` 错位 `i*55ms`、图标片 `bg-signal/12`、hover `-translate-y-0.5` + `shadow-2` + 模糊辉光）。过滤栏升级为 **sticky 玻璃命令条**（`backdrop-blur-md`）。大纲+卡片双模、全局统计、高级过滤、`jumpTo` 跳转逻辑全部保留。
- **AssetManager.tsx**：卡片 hover 拉升（`-translate-y-1` + `border-signal/30` + `shadow-2` + `ring-1`），分类栏加渐变高光条。网格/列表双视图、波形 canvas 播放器、棋盘格底衬、拖拽上传、refs 模态、云同步保留。
- **CharacterManager.tsx**：表情墙卡与花名册卡 hover 抛光（lift + ring + shadow）。主从布局、Expression Wall、TTS 语音预设（`TTS_VOICE_PRESETS`）、拖拽上传、默认表情配置保留。
