### 关键结论（已写报告）
- **P0 严重**：
  1. `evalCondition`(varRuntime.ts:53-73) 沙箱逃逸——白名单放行 `(`/`=`，`localStorage.clear()` 真执行。
  2. `reducer.applyDelta`(reducer.ts:89-93) 加载缺 `delta.audio` 即崩；`loadProjectData` 无校验。
  3. Ren'Py 导出变量名未消毒(rpyExporter.ts:336/1178)→非法 `default`/`$`；`displayName` 与 `escapeDialogue` 漏转义 `\n`(108-110)；`BUILTIN_TRANSITIONS`(118-130) 含非法/错名(glitter/squeeze*/facin/facout 非法；moveinup→moveintop 等方向后缀错)。*复核 Ren'Py 官方文档纠正子审查：flash/vpunch/hpunch 是真实内置 transition，保留。*
  4. `deleteAsset`(appStore.ts:331-332) 从不 `revokeObjectURL(blobUrl)` → Web 模式 blob 泄漏。
- **P1 体验**：变量监视 Dock 无法收成 rail(常驻288px, AppLayout.tsx:443)；右侧三 Dock 堆叠挤压舞台无 min-width。
- **P2 健壮**：AI `cleanup()` 死代码未调用致 180s 定时器泄漏(aiDirector.ts:846-849)；`parseDirective` 不容忍尾逗号/无部分恢复；模态无 focus trap。
- **良好**：Store 单一数据源(resolvedStates 同 set 派生)、流式 sw-asset 协议零 base64 驻留、overlay portal 架构、断流 30s/180s 看门狗、AI 单事务提交。
